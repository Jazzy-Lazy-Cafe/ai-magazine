# Podcast Summary Review Skill

## 목적
podcast-summary skill로 생성된 JSON 파일의 품질을 체계적으로 검수하여, 인터뷰 콘텐츠의 **유기적 연결성**과 **독자 친화성**을 평가하고 개선점을 제안합니다.

## 검수 대상
- 위치: `.claude/skills/podcast-summary/transcripts/json/*.json`
- 사용자가 특정 파일을 지정하면 해당 파일만 검수
- 파일 지정이 없으면 가장 최근 수정된 JSON 파일을 검수

---

## 검수 워크플로우

### Step 1: 구조 검증
1. JSON 스키마 일치 여부 확인
2. 필수 필드 존재 여부 확인:
   - `metadata` (layout, title, description, date)
   - `hero` (header, split/full)
   - `opening_insight` (label, text, translation)
   - `sections` (배열, 각 섹션에 number, title, question, answer)
   - `footer` (quote, meta_text)
3. 섹션 수 확인 (권장: 8-15개)

### Step 2: 유기적 연결성 검수 (30점)
각 섹션 간의 흐름을 분석하여 다음 연결 패턴을 확인:

#### 연결 패턴 유형
| 패턴 | 설명 | 예시 |
|------|------|------|
| 키워드 연결 | 이전 답변의 키워드가 다음 질문에 등장 | "GDP 성장" → "GDP 성장과 실업의 관계는?" |
| 인과관계 연결 | 이전 답변의 결과가 다음 질문의 원인 | "AI가 일자리를 대체한다" → "대체된 일자리에 대한 대응은?" |
| 시간순 연결 | 과거→현재→미래 흐름 | "지금까지의 발전" → "현재 상황" → "미래 전망" |
| 구체화 연결 | 추상적 개념에서 구체적 사례로 | "AI의 위험성" → "구체적으로 어떤 위험?" |

#### 검수 항목
- [ ] 섹션 1-2: opening_insight의 주제가 첫 질문과 연결되는가?
- [ ] 섹션 N→N+1: 이전 답변이 다음 질문을 자연스럽게 유도하는가?
- [ ] 마지막 섹션→footer: footer가 전체 논의를 수렴하는가?
- [ ] 주제 전환 시 맥락이 유지되는가?

### Step 3: 독자 친화성 검수 (30점)

#### 두괄식 답변 확인
- 각 answer의 첫 문장이 질문에 대한 직접적 답변인가?
- "~입니다", "~합니다"로 명확하게 시작하는가?

#### 전문용어 처리 확인
- answer에 등장하는 전문용어가 `knowledge_items`에 설명되어 있는가?
- 설명이 비전공자도 이해할 수 있는 수준인가?

#### 구체성 확인
- 답변에 숫자, 비유, 구체적 사례가 포함되어 있는가?
- 추상적 주장만 있는 답변이 없는가?

### Step 4: 내용 완결성 검수 (25점)

#### 핵심 인사이트 점검
- opening_insight가 인터뷰의 핵심을 대표하는가?
- 중요한 논점이 누락되지 않았는가?

#### 답변 균형 점검
- 답변 길이가 4-10문장 사이인가?
- 특별히 짧거나 긴 답변이 있다면 정당한 이유가 있는가?

#### 섹션 분량 점검
- 섹션 수가 적정한가? (8-15개 권장)
- 중복되는 내용이 있는가?

### Step 5: 구조/메타데이터 검수 (15점)

- title 형식: "[인터뷰이 이름 영문] : [주제]" 준수
- description: 인터뷰이 배경과 주제가 포함되어 있는가?
- interview_context: 인터뷰 배경이 2-3문장으로 설명되어 있는가?
- hero 정보: guest 정보가 정확한가?

## 점수 산출 기준

### 총점: 100점
| 카테고리 | 배점 | 평가 기준 |
|----------|------|-----------|
| 유기적 연결성 | 30점 | 섹션 간 자연스러운 흐름 |
| 독자 친화성 | 30점 | 두괄식, 전문용어 설명, 구체성 |
| 내용 완결성 | 25점 | 핵심 인사이트, 답변 균형 |
| 구조/메타데이터 | 15점 | 스키마 일치, 필드 완성도 |

### 등급 기준
| 등급 | 점수 | 설명 |
|------|------|------|
| S | 95-100 | 즉시 발행 가능 |
| A | 85-94 | 경미한 수정 후 발행 가능 |
| B | 70-84 | 일부 섹션 수정 필요 |
| C | 55-69 | 상당한 수정 필요 |
| D | 0-54 | 재작성 권장 |

## 검수 시 유의사항

### "친절한 글"의 정의
1. **독자 입장 고려**: AI에 관심은 있지만 전문가가 아닌 독자
2. **단계적 설명**: 복잡한 개념은 예시→정의→함의 순서로
3. **실행 가능한 인사이트**: "그래서 뭘 해야 하지?"에 답이 있어야 함

### "유기적 연결"의 핵심
1. 독자가 "왜 갑자기 이 질문이 나오지?"라고 느끼면 안 됨
2. 각 섹션이 이전 섹션을 기반으로 발전해야 함
3. 마지막에는 전체를 관통하는 메시지로 수렴해야 함

### 수정 제안 원칙
1. 문제만 지적하지 말고 구체적인 대안 제시
2. 현재 텍스트의 맥락을 유지하면서 개선
3. 과도한 수정보다는 최소한의 변경으로 효과 극대화

## 사용 방법

1. 검수할 JSON 파일 지정:
   ```
   /podcast-summary-review anthropic-ceo-summary.json
   ```

2. 가장 최근 파일 자동 검수:
   ```
   /podcast-summary-review
   ```

3. 특정 카테고리만 검수:
   ```
   /podcast-summary-review --focus=연결성 anthropic-ceo-summary.json
   ```
