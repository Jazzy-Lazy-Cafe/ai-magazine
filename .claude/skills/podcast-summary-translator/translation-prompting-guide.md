# Claude를 활용한 자연스러운 한국어 번역 가이드

## 이 파일의 목적

**이 파일은 Claude에게 번역을 요청할 때 어떻게 프롬프트를 작성해야 좋은 결과를 얻을 수 있는지 설명합니다.**

- **SKILL.md**: Claude가 무엇을 해야 하는지 (기술적 워크플로우)
- **이 파일**: Claude에게 어떻게 요청해야 하는지 (프롬프팅 전략)

번역 패턴의 상세 목록은 [natural-korean-guide.md](../podcast-summary-review/natural-korean-guide.md)를 참조하세요.

---

## 핵심 원칙

Claude에게 번역을 요청할 때 **"자연스럽게 번역해주세요"만으로는 부족합니다.**
구체적인 가이드라인과 예시를 제공해야 합니다.

## 🎯 효과적인 프롬프팅 전략

### 전략 1: 역할 + 제약 + 예시

```markdown
당신은 10년 경력의 한국어 원어민 번역가입니다.
번역은 번역처럼 들려서는 안 되며, 한국인이 자연스럽게 말하는 표현을 사용해야 합니다.

금지 사항:
- "~하는 것", "~에 대한" 같은 명사형 과다 사용
- "~에 의해" 같은 수동태
- "그것은", "이것은"으로 시작하는 문장
- 영어 문장 구조 그대로 유지

좋은 예시:
❌ "그것은 중요한 것입니다"
✅ "이건 중요합니다"

❌ "~라고 말하는 것은 새로운 것이 아닙니다"
✅ "새삼스러운 얘기는 아니지만"

이제 다음을 번역해주세요:
[원문]
```

### 전략 2: 단계별 검증 요청

번역 후 자가 검증을 요청하면 품질이 20-30% 향상됩니다.

```markdown
다음 영문을 한국어로 번역한 후, 검증 질문에 답해주세요.

원문:
[...]

번역:
[...]

검증:
1. 원문의 모든 문장이 번역되었나요?
2. 대시(—)나 세미콜론(;)으로 구분된 부분이 모두 포함되었나요?
3. 번역투 표현(~것, ~에 대한, ~에 의해)이 있나요?
4. 한국인이 실제로 이렇게 말하나요?
5. 더 자연스러운 표현이 있나요?
```

### 전략 3: Bad/Good 예시 쌍 제공

Claude는 패턴 학습에 강합니다. 나쁜 예시와 좋은 예시를 함께 보여주세요.

```markdown
번역 시 다음 패턴을 피하고 개선하세요:

❌ 명사형 과다:
"관심을 가지다" → ✅ "관심을 두다"
"어려움이 있다" → ✅ "어렵다"

❌ 수동태:
"~에 의해 작성되다" → ✅ "~가 쓰다"
"보여지다" → ✅ "보이다"

❌ 무생물 주어:
"태풍이 기차를 멈추게 했다" → ✅ "태풍 때문에 기차가 멈췄다"

❌ 메타 발화 직역:
"~라고 말하는 건 새로운 게 아니다" → ✅ "새삼스럽지 않지만"
"솔직히 말하면" → ✅ "솔직히" / "사실"

이제 번역:
[원문]
```

### 전략 4: 복합 문장 처리 가이드

영어의 긴 문장은 한국어로 여러 문장으로 나누는 게 자연스럽습니다.

```markdown
복합 문장 처리 규칙:
1. 대시(—): 별도 문장으로 분리
2. 세미콜론(;): 별도 문장으로 분리
3. 한 문장에 동사 3개 이상: 2-3개 문장으로 분리

예시:
원문: "AI will transform the economy—what seems political today will become universal because everyone will see the need."

❌ 나쁜 번역 (한 문장으로):
"오늘날 정치적으로 보이는 것도 모두가 필요성을 인식하면서 보편적이 될 것입니다"
→ 문제: 대시 앞부분("AI will transform the economy") 누락!

✅ 좋은 번역 (분리):
"AI가 경제를 변화시킬 것입니다. 지금은 정치적으로 보여도 모두가 필요성을 깨달으면서 보편적이 될 것입니다."
```

### 전략 5: 반복 개선 패턴

첫 번역 → 검토 → 개선을 반복하세요.

```markdown
[1단계] 초벌 번역
원문을 번역해주세요.

[2단계] 번역투 검수
다음 번역에서 번역투를 찾아주세요:
- "~것", "~에 대한" 과다 사용
- 수동태
- 명사형

[3단계] 개선
번역투를 제거한 개선 버전을 제시해주세요.
```

### 프롬프트에 포함할 핵심 지침

```markdown
특별히 주의할 표현:
1. 메타 발화: "~라고 말하는 건" → 접속사로 처리
2. 정치 표현: "politically coded" → "정치적 색깔을 띤"
3. 기술 용어: 영문 병기 (예: "Scaling Laws (스케일링 법칙)")
```

## 📋 완전한 프롬프트 템플릿

```markdown
# 역할 정의
당신은 10년 경력의 전문 한영 번역가입니다.
번역은 자연스러운 한국어로, 번역임을 느낄 수 없어야 합니다.

# 번역 원칙
1. 의미 보존: 원문의 모든 문장 번역
2. 자연스러움: 한국인이 실제로 쓰는 표현
3. 간결성: 불필요한 명사형, 조사 제거
4. 문장 분리: 대시(—), 세미콜론(;)은 별도 문장으로

# 금지 사항
- "~하는 것", "~에 대한" 과다 사용
- "~에 의해" 수동태
- "그것은", "이것은" 남발
- 영어 문장 구조 그대로 유지

# 패턴 가이드
다음 가이드의 120가지 패턴을 참조하세요:
[natural-korean-guide.md](../podcast-summary-review/natural-korean-guide.md)

주요 예시:
❌ "관심을 가지다" → ✅ "관심을 두다"
❌ "~라고 말하는 건 새로운 게 아니다" → ✅ "새삼스럽지 않지만"
❌ "정치적으로 코딩된" → ✅ "정치적 색깔을 띤"

# 원문
[여기에 원문 입력]

# 번역
[번역 결과]

# 검증
1. 모든 문장이 번역되었나요?
2. 번역투가 있나요?
3. 더 자연스러운 표현은?
```

## ⚠️ 흔한 실수

### 실수 1: 모든 문장을 번역하지 않음

```markdown
원문: "AI will change everything—we need to prepare now."

❌ "우리는 지금 준비해야 합니다"
→ 앞부분("AI will change everything") 누락!

✅ "AI가 모든 것을 바꿀 것입니다. 우리는 지금 준비해야 합니다."
```

### 실수 2: 긴 문장을 억지로 한 문장으로

```markdown
원문: "The company grew fast, revenue increased tenfold, and we hired 500 people."

❌ "회사는 빠르게 성장했고 매출은 10배 증가했으며 500명을 고용했습니다"
→ 너무 길고 숨이 참

✅ "회사는 빠르게 성장했습니다. 매출은 10배 증가했고, 500명을 고용했습니다."
```

### 실수 3: 맥락 없이 단어만 번역

```markdown
"run" → "달리다"? "운영하다"? "실행하다"?

반드시 문맥을 고려:
"run a company" → "회사를 운영하다"
"run a program" → "프로그램을 실행하다"
"run fast" → "빠르게 달리다"
```

## 🚀 고급 팁

### 팁 1: "왜?"를 함께 설명하기

```markdown
번역만 주지 말고 이유도 설명 요청:

"다음을 번역하고, 선택한 표현의 이유를 설명해주세요"

예:
번역: "새삼스러운 얘기는 아니지만"
이유: "I'm not saying anything novel"은 메타 발화로, 한국어에서는
접속어로 처리하는 게 자연스럽기 때문
```

### 팁 2: A/B 테스트

```markdown
같은 문장의 여러 번역안을 요청:

"다음 문장의 자연스러운 번역 3가지를 제시하고, 각각의 뉘앙스 차이를 설명해주세요"

예:
1. "새삼스러운 얘기는 아니지만" - 격식체, 객관적
2. "뻔한 얘기지만" - 구어체, 캐주얼
3. "다들 아는 얘기지만" - 친근함, 공감 유도
```

### 팁 3: 역번역 검증

```markdown
번역 → 영어로 역번역 → 원문과 비교

이 과정에서 의미 왜곡 발견 가능
```
